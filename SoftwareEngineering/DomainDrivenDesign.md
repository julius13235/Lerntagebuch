## Domain-Driven Design

### Principle of global architecture across software development processes. It describes three main principles:
* **Domain**
* **Application**
* **Infrastructure**

## Domain
### The domain describes the entities and their relationships. It defines the information that applications need to use. Tools and technologies used include:
* Structured Query Language (SQL)  
* Protobuf  
* Object-Relational Mapper (ORM)

## Application
### The application describes the logic applied to the entities. Tools and technologies used include:
* Programming languages: Java, C#  
* Pre-configured tools: Docker, Docker Compose

## Infrastructure
### Infrastructure implements the application and domain. It relies on concrete hardware or software setups, for example:
* PostgreSQL on Windows via Docker  
* Nginx on Linux  
* Apache2 on macOS
