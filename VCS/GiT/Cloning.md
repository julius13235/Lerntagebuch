## Cloning a repository
Cloning a repository from any source means that we copy the global software state into a local code repository.
 
## Global & Local Repository
 
1. Global Repository: The Server
 
The server holds all the information that the developer, administrator or product owner needs to know about various code, documentation, tests, CI/CD configurations and more. Anything that is required for the whole software release management cycle and development life cycle.
 
2. Local Repository: The Client
 
The client copies the global server repository into the local host machine.
 
This can be done with the command `git clone [url]`
