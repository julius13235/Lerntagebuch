# Issues

We all hate issues, but why do they exist? Think of them as a digital to-do list for your project.

### Creating an Issue
You can create an issue quickly using the GitHub CLI:
`gh issue create --title "Issue Title" --body "A description of the problem or task"`

Alternatively, you can simply go to the **Issues** tab on GitHub. 

### Managing Issues
Once created, you can see all issues and mark them as **Open**, **Completed**, or **Not Planned**. To keep things organized, you can:

*   **Labels:** Categorize them (e.g., as a "bug" or "feature request").
*   **Assignees:** Assign the task to a specific person.
*   **Projects & Milestones:** Link the issue to a specific project board or a milestone to track progress.

---

**Tip of the Prrrooosss:** If you want to save time, use `gh issue list` in your terminal to see everything that needs to be done without leaving your code.
